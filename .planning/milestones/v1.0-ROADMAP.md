# Roadmap: MIPS CPU Emulator

## Phase 1: Cross-Platform Fix & Code Cleanup
**Goal:** Fix compilation on non-Windows platforms and clean up obvious bugs and dead code.
**Mode:** mvp
**Success Criteria**:
1. `cargo build` succeeds on Linux with zero errors
2. `cargo test` passes all existing tests
3. `load_entry` no longer accumulates buffer contents between binary loads
4. No unused imports or dead code warnings on clean build
5. Unused `HW_MEM` global removed

## Phase 2: Module Refactor & Instruction Stubs
**Goal:** Rename the `mod` module to a conventional Rust name and implement remaining instruction stubs.
**Mode:** mvp
**Success Criteria**:
1. Module renamed from `r#mod` to `emu` (or similar) with all references updated
2. All R-type arithmetic/logic instructions implemented
3. All I-type immediate and memory instructions implemented
4. All J-type jump instructions implemented
5. All branch instructions implemented

## Phase 3: Integration & Validation
**Goal:** Validate the emulator against reference MIPS binaries and improve test coverage.
**Mode:** mvp
**Success Criteria**:
1. Emulator runs a simple MIPS test program end-to-end
2. Unit tests cover all implemented instructions
3. Memory access instructions handle alignment correctly
4. Debugger commands work reliably across platforms
